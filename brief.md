## 1. Цель проекта

### Задача

Создать Telegram Mini App — визуальный каталог услуг бьюти-мастера. Клиент открывает бота → нажимает кнопку → видит услуги с фотографиями и ценами → выбирает услугу → записывается. Всё внутри Telegram, без переходов на внешние сайты.

### Целевое действие пользователя

Приоритет 1 — **выбрать услугу и записаться** (через слот-пикер, автоподтверждение)
Приоритет 2 — **узнать цену** без необходимости писать мастеру

Конверсия считается состоявшейся, когда клиент подтвердил запись и получил уведомление.

### Метрики успеха MVP

| Метрика | Цель |
|---|---|
| Время загрузки Mini App | < 2 сек |
| Конверсия открытие → запись | > 15% |
| Возврат клиентов (повторная запись) | > 40% |
| NPS клиентов мастера | > 8 |

## 2. Точка входа

Клиент попадает в Mini App одним из способов:
1. Telegram-бот мастера → команда `/start` → кнопка «Открыть каталог»
2. Ссылка `t.me/BotName/app` (прямая ссылка на Mini App)
3. Кнопка в Telegram-канале мастера

**Бот:**
- Приветственное сообщение с именем мастера и фото
- Кнопка **«Открыть каталог»** — запускает Mini App
- Уведомления о подтверждении записи, напоминания

## 3. Экраны MVP

---

### Экран 1: Каталог услуг (главная)

**Цель:** Дать клиенту быстро найти нужную услугу и понять цену.

**Структура:**
- Шапка: аватар мастера + имя + специализация (например, «Nail-мастер · Москва»)
- Горизонтальный скролл категорий: «Все» / «Маникюр» / «Педикюр» / «Дизайн» и т.д.
- Карточки услуг (список или сетка 2 колонки):
  - Фото результата
  - Название услуги
  - Длительность (`⏱ 90 мин`)
  - Цена (`от 2 000 ₽` или точная)
  - Кнопка «Выбрать»
- MainButton скрыта на этом экране (или «Записаться» для быстрой записи)

**Обязательно:**
- Цена на каждой карточке — никогда не скрыта
- Фото — обязательно (плейсхолдер если не загружено)
- Категории — горизонтальный scroll, sticky под шапкой

---

### Экран 2: Детали услуги

**Цель:** Дать полную информацию и запустить запись.

**Структура:**
- BackButton (нативный Telegram)
- Фото галерея (свайп, точки-индикаторы)
- Название (H1)
- Бейджи: длительность + цена
- Описание (expandable если длинное)
- Блок «Мастер»: аватар + имя + рейтинг (опционально в MVP)
- MainButton: **«Записаться — 2 000 ₽»**

---

### Экран 3: Выбор даты и времени

**Цель:** Выбрать удобный слот без звонка мастеру.

**Структура:**
- BackButton
- Step indicator: «Шаг 2 из 3»
- Горизонтальный scroll дат (ближайшие 14 дней)
- Сетка временных слотов (доступные / недоступные)
- Сводка внизу: услуга + выбранный слот
- MainButton: **«Продолжить»** (активна только после выбора слота)

**Правила:**
- Недоступные слоты: серые, не кликабельны
- Прошедшие даты: не показываются
- MainButton задизейблена до выбора времени: `MainButton.disable()`

---

### Экран 4: Итог записи

**Цель:** Финальная проверка перед подтверждением.

**Структура:**
- BackButton
- «Ваша запись» заголовок
- Карточка сводки:
  - Услуга + фото превью
  - Мастер
  - Дата и время
  - Стоимость
- Контакт клиента (имя из Telegram, телефон если нужен)
- MainButton: **«Подтвердить запись»**

---

### Экран 5: Запись подтверждена

**Цель:** Позитивное подкрепление, следующие шаги.

**Структура:**
- Анимация успеха (чекмарк)
- «Запись подтверждена!»
- Карточка: дата, время, адрес мастера
- Кнопка «Добавить в календарь»
- Текст: «Мы напомним за 24 часа»
- MainButton: **«Вернуться в каталог»** или **«Закрыть»** (`WebApp.close()`)

**После закрытия Mini App:**
Бот отправляет сообщение в чат с подтверждением и кнопками «Отменить» / «Перенести».

---

### Экран 6: Мои записи

**Цель:** Управление записями без звонка мастеру.

**Структура:**
- BackButton
- Вкладки: «Предстоящие» / «Прошлые»
- Карточки записей: услуга + дата/время + статус
- Tap для раскрытия: полные детали + кнопка «Отменить»
- Пустое состояние: иллюстрация + «Запишитесь на первую процедуру» + CTA

## 5. Telegram-специфика (обязательно)

| Элемент | Реализация |
|---|---|
| MainButton | На каждом экране с основным действием. Контекстный текст. |
| BackButton | На всех экранах кроме Каталога (root). |
| HapticFeedback | `light` при выборе слота, `medium` при подтверждении, `success` на success-экране |
| Тема | Только CSS `--tg-theme-*` переменные. Никаких хардкод-цветов. |
| `expand()` | Вызывать при инициализации. |
| `ready()` | Вызывать после первого рендера. |
| CloudStorage | Хранить: последняя категория, выбранный мастер. |
| initDataUnsafe | Имя клиента для сводки записи. |

## 6. Дизайн-система

### Цвета (Telegram theme-адаптивные)

```css
/* Использовать только TG переменные */
background-color: var(--tg-theme-bg-color);
color: var(--tg-theme-text-color);
background-color: var(--tg-theme-secondary-bg-color); /* карточки */
color: var(--tg-theme-hint-color);                    /* вторичный текст */
color: var(--tg-theme-link-color);                    /* ссылки */
background-color: var(--tg-theme-button-color);       /* акцент */
color: var(--tg-theme-button-text-color);
```

### Типографика

| Элемент | Размер | Вес |
|---|---|---|
| Заголовок экрана | 20–22px | 600–700 |
| Название услуги в карточке | 15–16px | 500–600 |
| Цена | 16–18px | 700 |
| Вторичный текст | 13–14px | 400 |
| Длительность / бейдж | 12–13px | 400 |

## 7. Правила отображения цен

| Ситуация | Формат |
|---|---|
| Фиксированная цена | `2 000 ₽` |
| Диапазон | `от 2 000 ₽` |
| Скидка | `~~2 500 ₽~~ 2 000 ₽` |
| Пакет | `3 процедуры — 5 500 ₽` |

**Правило:** цена всегда видна на карточке. Никогда «цена по запросу» или пустое поле.

## 8. Уведомления через бот

| Событие | Сообщение | Кнопки |

## 9. Состояния загрузки и ошибок

| Ситуация | Реализация |
|---|---|
| Загрузка каталога | Skeleton-карточки (shimmer анимация) |
| Загрузка слотов | Skeleton-сетка |
| MainButton в процессе | `MainButton.showProgress()` |
| Нет сети | Snackbar: «Нет соединения» + кнопка «Повторить» |
| Слот занят (гонка) | Inline сообщение: «Это время уже занято» + прокрутка к следующему |
| Пустая категория | Иллюстрация + «Скоро появятся новые услуги» |

## 10. Антипаттерны — запрещено

| Запрещено | Причина |
|---|---|
| Экран логина/регистрации | Telegram уже авторизован |
| Редирект на внешний сайт | Ломает UX, высокий отказ |
| Скрытые цены | Главная причина ухода |
| Кастомная кнопка «Назад» вместо нативной | Конфликт с Telegram BackButton |
| `alert()` / `confirm()` браузерные | Не работают в Telegram WebView |
| Хардкод цветов | Ломается в тёмной теме |
| Ожидание ответа мастера для подтверждения | Главная жалоба пользователей |
