# brief.md — Техническое задание

> Версия: MVP 1.0 · Март 2026
> Связанные файлы: [project.md](project.md) · [research.md](research.md) · [CLAUDE.md](CLAUDE.md) · [PLAN.md](PLAN.md)

---

## 1. Цель

Telegram Mini App — визуальный каталог услуг бьюти-мастера.

**Главный сценарий:**
Клиент открывает бота → нажимает «Открыть каталог» → видит услуги с фото и ценами → выбирает услугу → записывается. Всё внутри Telegram. Никаких переходов на сайт.

**Целевое действие:** клиент подтвердил запись и получил уведомление в боте.

---

## 2. Точка входа

- Telegram-бот → команда `/start` → приветственное сообщение + кнопка **«Открыть каталог»**
- Прямая ссылка `t.me/BotName/app`
- Кнопка в канале мастера

## 3. Экраны MVP

---

### Экран 1 — Каталог услуг (главная, root)

**Что видит пользователь:**
- Шапка: аватар мастера + имя + специализация (`Nail-мастер · Москва`)
- Горизонтальный скролл категорий: `Все` / `Маникюр` / `Педикюр` / `Дизайн` / `Уход`
- Карточки услуг (вертикальный список):
  - Фото результата (16:9, скруглённые углы 16px)
  - Название услуги
  - Длительность (`⏱ 90 мин`)
  - Цена (`от 2 000 ₽` или точная)
  - Кнопка **«Выбрать»** (высота ≥ 44px)

**Действия пользователя:**
| Действие | Что происходит |
|---|---|
| Тап на категорию | Фильтрация карточек на месте, без перехода |
| Тап на карточку услуги | Переход на Экран 2 |
| Тап **«Выбрать»** | Переход на Экран 2 |

**Telegram-компоненты:**
- BackButton: **скрыта** (root-экран)
- MainButton: **скрыта**
- HapticFeedback: `light` при тапе на категорию и карточку

**Состояния:**
- Загрузка → Skeleton-карточки (shimmer)
- Пустая категория → иллюстрация + «Скоро появятся услуги»
- Нет сети → snackbar «Нет соединения · Повторить»

---

### Экран 2 — Детали услуги

**Что видит пользователь:**
- Галерея фото (свайп + точки-индикаторы внизу)
- Название услуги (H1, 20–22px, вес 700)
- Бейджи: `⏱ 90 мин` и `2 000 ₽` (крупно, вес 700)
- Описание (2–3 строки, expandable по «Читать дальше»)
- Блок мастера: аватар + имя + `★ 4.9`

**Действия пользователя:**
| Действие | Что происходит |
|---|---|
| Свайп по галерее | Следующее/предыдущее фото |
| «Читать дальше» | Разворачивается полный текст описания |
| BackButton (нативный) | Назад на Каталог |
| MainButton **«Записаться — 2 000 ₽»** | Переход на Экран 3 |

**Telegram-компоненты:**
- BackButton: **показана**
- MainButton: **«Записаться — 2 000 ₽»**, всегда активна
- HapticFeedback: `medium` при нажатии MainButton

---

### Экран 3 — Выбор даты и времени

**Что видит пользователь:**
- Индикатор шага: `Шаг 1 из 2`
- Горизонтальный скролл дат (ближайшие 14 дней):
  - День недели (`Пн`)
  - Число (`24`)
  - Точка если есть слоты
- Сетка временных слотов:
  - Доступный: активная кнопка (≥ 44px)
  - Недоступный: серый, не кликабелен
- Сводка внизу: название услуги + выбранная дата/время

**Действия пользователя:**
| Действие | Что происходит |
|---|---|
| Тап на дату | Загрузка слотов этого дня |
| Тап на слот | Выделение слота, активация MainButton |
| BackButton | Назад на Детали услуги |
| MainButton **«Продолжить»** | Переход на Экран 4 |

**Telegram-компоненты:**
- BackButton: **показана**
- MainButton: **«Продолжить»**, `disable()` до выбора слота
- HapticFeedback: `light` при выборе даты и слота

**Состояния:**
- Выбор даты → skeleton-сетка слотов
- Слот уже занят (гонка) → inline: «Время занято» + прокрутка к следующему свободному

---

### Экран 4 — Итог записи

**Что видит пользователь:**
- Заголовок «Ваша запись»
- Карточка сводки:
  - Мини-фото услуги + название
  - Дата и время (крупно)
  - Стоимость (крупно, вес 700)
- Имя клиента (из `initDataUnsafe.user.first_name`)
- Телефон (если уже давал, иначе кнопка «Добавить телефон»)

**Действия пользователя:**
| Действие | Что происходит |
|---|---|
| BackButton | Назад на выбор даты/времени |
| «Добавить телефон» | Нативный запрос контакта Telegram |
| MainButton **«Подтвердить запись»** | API-запрос → Экран 5 |

**Telegram-компоненты:**
- BackButton: **показана**
- MainButton: **«Подтвердить запись»**, при нажатии → `showProgress()`
- HapticFeedback: `medium` при нажатии MainButton

---

### Экран 5 — Запись подтверждена

**Что видит пользователь:**
- Анимация: чекмарк (CSS-анимация, без сторонних библиотек)
- Текст «Запись подтверждена!»
- Карточка: дата · время · адрес мастера
- Кнопка «Добавить в календарь» (≥ 44px, inline)
- Текст «Напомним за 24 часа в этом чате»

**Действия пользователя:**
| Действие | Что происходит |
|---|---|
| «Добавить в календарь» | Открывает нативный календарь с событием |
| BackButton | **Скрыта** (успех — конец флоу) |
| MainButton **«Вернуться в каталог»** | Возврат на Экран 1, стек сбрасывается |

**После закрытия Mini App:**
Бот автоматически отправляет в чат сообщение-подтверждение с кнопками «Отменить запись» / «Перенести».

**Telegram-компоненты:**
- BackButton: **скрыта**
- MainButton: **«Вернуться в каталог»**
- HapticFeedback: `notificationOccurred('success')`

---

### Экран 6 — Мои записи

**Что видит пользователь:**
- Вкладки: `Предстоящие` / `Прошлые`
- Карточки записей:
  - Название услуги
  - Дата и время
  - Статус-бейдж (`Подтверждено` / `Ожидает` / `Отменено`)
- Пустое состояние: иллюстрация + «Ещё нет записей» + кнопка «Записаться»

**Действия пользователя:**
| Действие | Что происходит |
|---|---|
| Тап на вкладку | Переключение Предстоящие/Прошлые |
| Тап на карточку | Раскрытие: полные детали + кнопка «Отменить» |
| «Отменить запись» | Confirm через `Telegram.WebApp.showConfirm()` → отмена |
| «Записаться снова» | Переход на Экран 2 с той же услугой |
| BackButton | Назад (если зашли не с главной) |

**Telegram-компоненты:**
- BackButton: **показана** (если не root)
- MainButton: **скрыта**

## 5. Навигация

### Нижняя панель (Bottom Tab Bar — постоянная)

```
[  Услуги  ]    [  Записи  ]    [  Мастер  ]
  (иконка)       (иконка)        (иконка)
```

Активная вкладка выделена акцентным цветом (`--tg-theme-button-color`).

### Полная карта переходов

```
Бот → кнопка «Открыть каталог»
  └── [1] Каталог (root)
        ├── Тап категории → фильтр на месте
        └── Тап карточки → [2] Детали услуги
                              └── MainButton → [3] Дата и время
                                                  └── MainButton → [4] Итог записи
                                                                        └── MainButton → [5] Подтверждено
                                                                                              └── MainButton → [1] Каталог

Нижняя панель → [6] Мои записи
  └── «Записаться снова» → [2] Детали услуги → ...
```

## 5. Дизайн-система

### Принципы
- Стиль: **нативный Telegram** — не сайт, не лендинг
- Визуала больше, текста меньше
- Все кнопки: высота **≥ 44px**, borderRadius 12px
- Скруглённые карточки: 16–20px

### Цвета — только CSS-переменные

```css
/* Фон */
background-color: var(--tg-theme-bg-color);
background-color: var(--tg-theme-secondary-bg-color); /* карточки, секции */

/* Текст */
color: var(--tg-theme-text-color);
color: var(--tg-theme-hint-color);   /* вторичный: длительность, подписи */
color: var(--tg-theme-link-color);

/* Акцент */
background-color: var(--tg-theme-button-color);
color: var(--tg-theme-button-text-color);
```

**Никаких хардкод-цветов** (`#ffffff`, `#000`, `pink` и т.д.) — ломается в тёмной теме.

### Типографика

| Элемент | Размер | Вес |
|---|---|---|
| Заголовок экрана | 20–22px | 600–700 |
| Название услуги в карточке | 15–16px | 500–600 |
| Цена | 16–18px | **700** |
| Вторичный текст | 13–14px | 400 |
| Бейдж / длительность | 12–13px | 400 |

### Анатомия карточки услуги

```
┌──────────────────────────────────────────┐
│  [Фото результата — 16:9, radius 16px]   │
│                                          │
│  Маникюр с гель-лаком          ⏱ 90 мин │
│  Аппаратная обработка + TopCoat          │
│                              от 2 000 ₽  │
│                         [  Выбрать  →  ] │  ← ≥44px
└──────────────────────────────────────────┘
```

## 6. Правила отображения цен

| Ситуация | Формат |
|---|---|
| Фиксированная | `2 000 ₽` |
| Диапазон | `от 2 000 ₽` |
| Скидка | `~~2 500 ₽~~ 2 000 ₽` |
| Пакет | `3 процедуры — 5 500 ₽` |

**Правило:** цена всегда видна. Никогда «цена по запросу» или пустое поле.

## 7. Уведомления через бот

| Событие | Сообщение | Кнопки |
|---|---|---|
| Запись создана | «✅ Запись подтверждена! [услуга, дата, время]» | — |
| За 24 часа | «⏰ Напоминаем: завтра в [время] — [услуга]» | Подтвердить · Перенести · Отменить |
| За 2 часа | «📍 Сегодня в [время] — [услуга]. Адрес: [адрес]» | — |
| Отмена | «Запись отменена. Записаться снова?» | Открыть каталог |

## 8. Telegram-компоненты — обязательные правила

| Компонент | Правило |
|---|---|
| `WebApp.expand()` | Вызывать первым при инициализации |
| `WebApp.ready()` | Вызывать после первого рендера |
| MainButton | На каждом экране с основным действием. Контекстный текст с ценой где возможно. |
| MainButton.disable() | Пока форма/выбор не заполнены |
| MainButton.showProgress() | Во время API-запросов |
| MainButton.hide() | На экранах без основного действия |
| BackButton.show() | На всех экранах кроме root (Каталог) и Успеха |
| BackButton.hide() | На Каталоге и Success-экране |
| HapticFeedback | `light` — тап карточки/слота; `medium` — переход к шагу; `success` — экран успеха; `error` — ошибка |
| CloudStorage | Хранить: последняя категория, последний просмотренный мастер |
| showAlert() | Для критичных ошибок (вместо `window.alert`) |
| showConfirm() | Для подтверждения отмены записи |

## 9. Состояния и ошибки

| Ситуация | Реализация |
|---|---|
| Загрузка каталога | Skeleton-карточки (shimmer, 3–4 штуки) |
| Загрузка слотов | Skeleton-сетка |
| MainButton в процессе | `showProgress(true)` + `disable()` |
| Нет сети | Snackbar снизу: «Нет соединения · Повторить» |
| Слот занят (гонка) | Inline под сеткой: «Время уже занято» + прокрутка к следующему |
| Ошибка API | `showAlert()` с текстом + предложение написать мастеру |
| Пустая категория | Иллюстрация + «Скоро появятся новые услуги» |
| Нет записей | Иллюстрация + «Запишитесь на первую процедуру» + CTA |

## 10. Чего НЕ будет в первой версии (v2+)

Следующие функции **исключены из MVP** и могут быть добавлены позже:

| Функция | Причина переноса |
|---|---|
| Онлайн-оплата внутри TMA | Требует подключения эквайринга, усложняет флоу |
| Система отзывов и рейтингов | Нужна история записей; MVP только собирает данные |
| Программа лояльности / баллы | Отдельная бизнес-логика, не нужна для первого запуска |
| Поиск по каталогу | На 5–15 услугах не нужен; появится при росте |
| Выбор мастера (если один мастер) | Если бот для одного мастера — лишний шаг |
| Акции и промокоды | Не влияют на MVP-конверсию |
| Многоязычность | Аудитория — русскоязычная |
| Мультилокация | Один мастер — одна точка |
| Чат с мастером внутри TMA | Есть нативный Telegram-чат |
| Push-уведомления о новых услугах | Рассылки через бот — фаза 2 |
| Авторизация / пароль | Telegram уже авторизовал пользователя |
| Экран onboarding / туториал | Интерфейс должен быть очевидным без обучения |

## 11. Антипаттерны — запрещено

| Запрещено | Почему |
|---|---|
| Редирект на внешний сайт | Ломает UX, высокий отказ |
| Скрытые цены | Главная причина ухода |
| Кастомная кнопка «Назад» в шапке | Конфликт с нативным BackButton |
| `window.alert()` / `window.confirm()` | Не работают в Telegram WebView |
| Хардкод цветов | Ломается в тёмной теме |
| Экран логина/регистрации | Telegram уже знает пользователя |
| Ожидание ответа мастера вручную | Главная жалоба — записаться и ждать 2 часа |
| Кнопки меньше 44px | Промахи, плохой UX на мобильном |
| Текстовый список услуг без фото | Бьюти — визуальная ниша |
| Splash-экран с анимацией | Telegram уже показывает загрузку; двойное ожидание |
