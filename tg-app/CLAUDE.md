# CLAUDE.md — Структура приложения tg-app/

Это Telegram Mini App — визуальный каталог услуг бьюти-мастера.
Стек: HTML + CSS + Vanilla JavaScript (без фреймворков).

---

## Файлы и их назначение

| Файл | Что делает |
|---|---|
| `index.html` | Точка входа. Подключает Telegram SDK, стили и скрипты. Содержит нижнюю навигацию. |
| `css/app.css` | Вся дизайн-система: переменные, компоненты, экраны, анимации. |
| `js/data.js` | Данные: мастер, услуги, категории, генерация слотов. Здесь меняются реальные данные. |
| `js/tg.js` | Обёртка над Telegram WebApp API: MainButton, BackButton, HapticFeedback. |
| `js/app.js` | Роутер + все экраны. Основная логика приложения. |

---

## Экраны и навигация

```
Каталог (root)
  └── Детали услуги
        └── Выбор даты/времени
              └── Итог записи
                    └── Запись подтверждена → Каталог (сброс стека)

Нижняя навигация:
  [Услуги] → Каталог
  [Записи] → Мои записи
  [Мастер] → Профиль мастера
```

## Где менять данные

### Данные мастера (имя, адрес, описание)
→ `js/data.js`, объект `MASTER`

### Список услуг
→ `js/data.js`, массив `SERVICES`
- Каждая услуга: id, category, emoji, photoGradient, title, shortDesc, fullDesc, duration, price, priceFrom

### Категории фильтров
→ `js/data.js`, массив `CATEGORIES`

### Рабочие часы и расписание слотов
→ `js/data.js`, функция `generateSlots(date)`
- startHour — начало рабочего дня
- endHour — конец рабочего дня
- busyPattern — какие слоты заняты (для демо)

### Цвета и стили
→ `css/app.css`
- В начале файла — CSS-переменные (:root)
- Цвета используются только через `var(--tg-theme-*)` — никакого хардкода

---

## Правила разработки

- **Один промт — одна задача.** Маленькие точные правки дают предсказуемый результат.
- **Цвета только через CSS-переменные** `--tg-theme-*` — поддержка тёмной темы.
- **Кнопки минимум 44px** в высоту (`--btn-min-height`).
- **BackButton** управляется через `TelegramAPI.showBackButton()` / `hideBackButton()`.
- **MainButton** управляется через `TelegramAPI.showMainButton()` / `hideMainButton()`.
- **HapticFeedback** вызывается при каждом значимом действии.

## Как запустить локально

Открыть `index.html` двойным кликом в браузере.
Или запустить локальный сервер (рекомендуется):

```bash
# Через Python
python -m http.server 3000

# Или через Node.js
npx serve .
```

Затем открыть http://localhost:3000

## Структура объекта App (app.js)

```javascript
App = {
  stack: [],        // Стек экранов для навигации назад
  booking: {},      // Данные текущей записи
  bookings: [],     // Список всех записей

  push(screen, data)      // Перейти на экран
  pop()                   // Назад
  replace(screen, data)   // Заменить текущий экран
  goRoot(screen)          // Вернуться на корневой экран
  goTab(tab)              // Переключить вкладку

  screens: {
    catalog: {}     // Экран 1 — Каталог
    service: {}     // Экран 2 — Детали услуги
    datetime: {}    // Экран 3 — Выбор даты/времени
    summary: {}     // Экран 4 — Итог записи
    success: {}     // Экран 5 — Успех
    bookings: {}    // Экран 6 — Мои записи
    master: {}      // Экран 7 — Профиль мастера
  }
}
```
