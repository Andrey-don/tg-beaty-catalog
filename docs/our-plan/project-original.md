## 1. Что это за проект

Telegram Mini App — каталог услуг бьюти-мастера. Клиент открывает Telegram-бота мастера, нажимает кнопку «Открыть» → попадает в Mini App, где видит услуги с фотографиями, ценами и кнопкой «Выбрать». Цель — дать мастеру персональный брендированный инструмент продаж прямо внутри Telegram, без перехода на внешние сайты.

---

## 2. Для кого

**Пользователь продукта (мастер):** nail-мастер, lash-мастер, бровист, парикмахер, массажист — любой бьюти-специалист, работающий самостоятельно или в небольшой команде.

**Конечный клиент (клиент мастера):**
- Уже знаком с мастером через Instagram, ВКонтакте или Telegram-канал
- Хочет посмотреть услуги и цены без необходимости писать в личку
- Привык к Telegram, не хочет устанавливать приложение

**Портрет клиента мастера:**
- Женщина 20–45 лет
- Постоянный клиент или по рекомендации
- Ценит скорость и удобство — хочет всё в одном месте
- Не хочет ждать ответа мастера, чтобы узнать цену

---

## 3. Проблема, которую решает

Большинство бьюти-мастеров ведут запись через переписку в Telegram или Instagram. Клиент пишет «сколько стоит маникюр?» — мастер отвечает через 2 часа. Существующие решения (YCLIENTS, Dikidi) либо слишком корпоративные и безличные, либо не дают полноценного каталога с фотографиями прямо в Telegram. Самостоятельно собранные боты на Salebot — только текст, без визуального каталога.

---

## 4. Основное позиционирование

> **«Твой персональный каталог услуг в Telegram — с фото, ценами и записью в один клик»**

**Дифференциаторы:**
- Визуальные карточки каждой услуги с фото результата
- Личный бренд мастера: имя, фото, стиль — не обезличенный шаблон
- Всё внутри Telegram — без редиректов на внешние сайты
- Мгновенное отображение цен — клиент не пишет «а сколько стоит?»

## 6. MVP — что входит в первую версию

| # | Экран | Содержимое |
|---|---|---|
| 1 | **Каталог (главная)** | Категории + карточки услуг с фото, названием, ценой |
| 2 | **Детали услуги** | Фото, описание, длительность, цена, кнопка «Выбрать» |
| 3 | **Выбор даты и времени** | Слот-пикер с доступными временными окнами |
| 4 | **Итог записи** | Сводка перед подтверждением |
| 5 | **Запись подтверждена** | Success-экран + напоминание через Telegram |
| 6 | **Мои записи** | Предстоящие и прошлые, кнопка отмены |

## 8. Конкуренты

| Конкурент | Чем отличаемся |
|---|---|
| YCLIENTS | Корпоративный iFrame, нет фото услуг, нет личного бренда |
| Dikidi | Обезличен, фото услуг необязательны, нет полного TMA |
| Salebot/BotHelp | Только текст, нет визуального каталога, ручное подтверждение |
| Taplink | Витрина без реальной записи, редирект в личку |

**Наше преимущество:** единственный инструмент с визуальным каталогом + автозаписью + личным брендом мастера — всё нативно внутри Telegram.

## 9. Технологии

### Ядро

| Слой | Технология | Почему |
|---|---|---|
| Фронтенд | **React + TypeScript** | Компонентный подход, экосистема TMA |
| Стили | **Tailwind CSS** | Быстро, консистентно, mobile-first |
| TMA SDK | **@telegram-apps/sdk** | Официальный SDK для Telegram Mini Apps |
| Роутинг | **React Router** | SPA-навигация с BackButton |
| Бэкенд | **Node.js + Express** | API для данных мастера и записей |
| База данных | **PostgreSQL** | Услуги, записи, клиенты |
| Бот | **Grammy / Telegraf** | Telegram Bot для точки входа |
| Деплой | **Railway / Render** | Бесплатный старт, простой деплой |

### Telegram-специфика

- `Telegram.WebApp.MainButton` — основная кнопка действия
- `Telegram.WebApp.BackButton` — навигация назад
- `Telegram.WebApp.HapticFeedback` — тактильный отклик
- CSS-переменные `--tg-theme-*` — поддержка светлой/тёмной темы

## 10. Стиль дизайна

**Направление: тёплый минимализм / бьюти-эстетика**

| Параметр | Значение |
|---|---|
| Палитра | Тёплые нейтральные тона: беж, кремовый, пыльно-розовый |
| Акцент | Один контрастный цвет (настраивается под бренд мастера) |
| Карточки | Скруглённые углы 16–20px, тень, фото в полную ширину |
| Типографика | Чистая, читаемая. Inter или системный шрифт |
| Цены | Всегда крупно и заметно — никогда не скрыты |
| Настроение | Профессионально, но тепло — не корпоративно |

---

## 11. Чего не будет на старте

- Регистрация/логин — Telegram уже знает пользователя
- Оплата онлайн (v2)
- Мультиязычность (v2)
- Несколько мастеров / студия (v2)
- Блог или контент-раздел
- Push-уведомления через внешний сервис (используем Telegram)
